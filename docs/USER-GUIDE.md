# คู่มือการใช้งานระบบควบคุมเอกสาร (DCC)
**Document Control Center — บริษัท ศ.วรภัทร อินเตอร์ ฟู้ดส์ จำกัด**
อ้างอิงระเบียบปฏิบัติ **QP-DC-01 การควบคุมเอกสาร** · ฉบับปรับปรุง (มิ.ย. 2026)

---

## 1. ภาพรวมระบบ
ระบบบริหารคำร้องและทะเบียนเอกสารคุณภาพแบบดิจิทัล ครอบคลุมตั้งแต่การยื่นขอจัดทำ/แก้ไข/ยกเลิกเอกสาร → อนุมัติ → ขึ้นทะเบียน → แจกจ่าย → ติดตามรอบทบทวน พร้อมแจ้งเตือนอีเมลอัตโนมัติ

| รายการ | ลิงก์ |
|---|---|
| 🖥️ หน้าหลัก (QA / DCC) | `https://dcs-document-control.swifoods.workers.dev` |
| 📝 หน้ายื่นคำร้อง (หน่วยงานอื่น) | `…/submit` |
| 🖨️ พิมพ์ใบ DAR | `…/dar/<เลขคำร้อง>` |

**แบบฟอร์มที่เกี่ยวข้อง:** DAR = FM-MR-01 · บัญชีแม่บท (MDL) = FM-MR-02 · บันทึกแจกจ่าย/เรียกคืน = FM-MR-03

## 2. การติดตั้งลงเครื่อง (PWA)
ใช้ได้ทันทีผ่านเบราว์เซอร์ หรือจะ "ติดตั้ง" เป็นแอปก็ได้:
- **Windows/Mac (Chrome/Edge):** เปิดหน้าหลัก → กดไอคอน **⊕ Install** ที่แถบ URL → แอปเปิดเป็นหน้าต่างแยก มีไอคอนบนเดสก์ท็อป
- **Android (Chrome):** เมนู ⋮ → "เพิ่มลงในหน้าจอหลัก"
- **iPhone (Safari):** ปุ่มแชร์ → "เพิ่มลงในหน้าจอโฮม"

## 3. การเข้าสู่ระบบ และบทบาท
- เปิดหน้าหลัก → ใส่ **รหัสผ่าน** (เริ่มต้น: `Swifoods@2026`) → ระบบจำไว้ในเครื่อง
- หน้า `/submit` **ไม่ต้องใช้รหัส** (สำหรับหน่วยงานทั่วไป)

| บทบาท | สิทธิ์ |
|---|---|
| **หน่วยงานเจ้าของเอกสาร (ผู้จัดทำ)** | ยื่นคำร้องผ่าน `/submit` |
| **ผู้ทบทวน (หัวหน้าแผนก / MR)** | ตรวจทานคำร้อง (หน้าหลัก) |
| **ผู้อนุมัติ (MD)** | อนุมัติ/ปฏิเสธ (หน้าหลัก) |
| **DCC (เจ้าหน้าที่ควบคุมเอกสาร)** | ขึ้นทะเบียน MDL, แนบฉบับเต็ม, แจกจ่าย, ติดตามทบทวน |

## 4. ขั้นตอนการใช้งานตามวงจรเอกสาร

### 4.1 ยื่นคำร้อง (DAR · FM-MR-01)
**หน่วยงานอื่น:** เปิด `…/submit` — **QA/DCC:** เมนู "สร้างคำร้อง"
1. กรอก **ผู้ขอ + อีเมล + แผนก** (อีเมลใช้รับแจ้งผล)
2. เลือก **ประเภทคำขอ:** จัดทำใหม่ / ปรับปรุง-แก้ไข / ยกเลิก / ขอสำเนา-ขอใช้
3. กรอก **ข้อมูลเอกสาร:** ประเภท (QM/QP/WI/FM/SD), ชื่อเอกสาร, รหัส
   - 🔢 กด **"ออกเลขอัตโนมัติ"** → ระบบให้รหัสรันถัดไป (รูปแบบ `ประเภท-แผนก-ลำดับ` เช่น `QP-QA-03`)
   - 📎 กด **"อัปโหลดไฟล์"** เพื่อแนบ **ไฟล์ฉบับร่าง** (หรือวางลิงก์)
4. กรอก **เหตุผล + วันที่คาดว่าจะแล้วเสร็จ + ผู้อนุมัติ**
5. กด **ส่งคำร้อง** → ได้เลขคำร้อง `REQ-…`

> ทันทีที่ส่ง ระบบจะ **อีเมลแจ้ง DCC** (qa.sorworaphat@gmail.com) ให้พิจารณา

### 4.2 พิจารณา / อนุมัติ
เมนู **"คำร้องทั้งหมด"** → แตะคำร้อง:
- ดูรายละเอียด + เปิดไฟล์ฉบับร่าง
- ใส่ **ผู้ตัดสิน + ความเห็น** → กด **✅ อนุมัติ** หรือ **❌ ปฏิเสธ**
- 🖨️ กด **"พิมพ์ใบ DAR"** เพื่อพิมพ์แบบฟอร์ม(มีช่องลงนาม 3 ขั้น: ผู้จัดทำ → ผู้ทบทวน(MR) → ผู้อนุมัติ(MD))

> เมื่อตัดสินแล้ว ระบบ **อีเมลแจ้งผลกลับผู้ขอ** (ถ้ากรอกอีเมลไว้)

### 4.3 ขึ้นทะเบียน + แนบเอกสารฉบับเต็ม (MDL · FM-MR-02)
- **อนุมัติแล้ว = ระบบขึ้นทะเบียนใน MDL ให้อัตโนมัติ** (Status = Active, Rev 00, ตั้งวันทบทวน +1 ปี)
  - ถ้าเป็นคำขอ "ยกเลิก" → ตั้งสถานะเอกสารเป็น **Obsolete** ให้
- เมนู **"MDL"** → เปิดเอกสาร → ที่ช่อง **"เอกสารฉบับเต็ม (ฉบับอนุมัติ)"** กด **📎 อัปโหลดเอกสารฉบับเต็ม** (เป็นหน้าที่ของ DCC ตาม QP — ผู้จัดทำส่งต้นฉบับให้ DCC)
- แก้ไขรายละเอียด/Rev/วันบังคับใช้/วันทบทวนได้ในหน้านี้

### 4.4 แจกจ่าย / เรียกคืน (FM-MR-03)
เมนู **"แจกจ่ายเอกสาร"** → กด **"+ แจกจ่าย"** (หรือกดปุ่ม 📦 จากคำร้องที่อนุมัติ / จากเอกสารใน MDL):
- บันทึก: รหัสเอกสาร, Rev, **ผู้ถือครอง**, แผนก, **ชุดที่ (Copy No.)**, ประเภท (**Controlled / Uncontrolled**), วันแจกจ่าย
- เมื่อเรียกคืน → เปลี่ยนสถานะเป็น **Returned** + ใส่วันที่เรียกคืน

### 4.5 ติดตามรอบทบทวน (Expiry Monitor)
เมนู **"Expiry Monitor"** — แสดงเอกสารเรียงตามรอบทบทวน:
- 🔴 เลยกำหนด · 🟡 ใกล้ครบ (≤30 วัน) · 🟢 ปกติ
- ทบทวนอย่างน้อยปีละ 1 ครั้ง (QP-DC-01) → ถึงรอบให้เริ่มออก DAR ใหม่

### 4.6 Dashboard
ภาพรวม: เอกสารทั้งหมด · รออนุมัติ · อนุมัติแล้ว · ปฏิเสธ + คำร้องล่าสุด

## 6. การดูแลระบบ (สำหรับผู้ดูแล)
- **เปลี่ยนรหัสผ่านหน้าแอดมิน:** Cloudflare → Workers & Pages → `dcs-document-control` → Settings → Variables → เพิ่ม **`ADMIN_KEY`** = รหัสใหม่
- **เปลี่ยนอีเมลผู้รับแจ้งเตือน:** ตั้งตัวแปร **`NOTIFY_EMAIL`** (หรือแก้ในโค้ด)
- **โครงสร้างระบบ:** Cloudflare Worker (โค้ด+หน้าเว็บ) · D1 `dcs_document_control` (ข้อมูล) · R2 `dcs-document-files` (ไฟล์แนบ) · Apps Script (ส่งเมล)
- **อัปเดตระบบ:** push โค้ดขึ้น GitHub → Deploy อัตโนมัติ

## 7. แก้ปัญหาเบื้องต้น
| อาการ | วิธีแก้ |
|---|---|
| ไม่เห็นปุ่ม Install | ใช้ Chrome/Edge, เปิดผ่าน https, refresh แรง ๆ |
| เข้าหน้าหลักแล้วขึ้นหน้าใส่รหัส | ปกติ — ใส่รหัสผ่าน |
| อีเมลไม่เข้า | ตรวจ Spam / ตรวจ Apps Script (Executions) ว่ามี doPost |
| อัปโหลดไฟล์ไม่ได้ | ไฟล์ต้อง ≤ 25MB |
| หน้าเก่าไม่อัปเดต | ปิด-เปิดแอป/แท็บใหม่ (ล้าง cache) |
